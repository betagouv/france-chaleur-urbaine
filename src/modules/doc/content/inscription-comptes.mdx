# Inscription et comptes

Cette page décrit comment les comptes utilisateurs sont créés et activés, ce qui se passe à la connexion, les emails envoyés et les données modifiées à chaque étape.

## Vue d'ensemble

Un compte peut être créé de trois façons. Seule l'inscription publique passe par une confirmation d'email ; les deux autres parcours produisent un compte immédiatement valide.

Les nœuds colorés sont cliquables et mènent au modèle d'email ou à l'activité correspondante.

## Les rôles

| Rôle | Qui | Comment le compte est créé |
| --- | --- | --- |
| Particulier | Citoyen qui teste son adresse et suit ses demandes | Inscription publique ou admin |
| Professionnel | Bureau d'études, bailleur, syndic… | Inscription publique ou admin |
| Gestionnaire | Exploitant de réseau (ENGIE, Dalkia…) — voir [Rôle : gestionnaire](/admin/doc/role-gestionnaire) | Admin ou API partenaire |
| Collectivité | Commune ou EPCI qui suit son territoire | Admin |
| ALEC | Agence locale de l'énergie et du climat | Admin |
| CCRT | Chargé de mission compétent sur un territoire | Admin |
| Admin | Équipe France Chaleur Urbaine | Admin |

Gestionnaire, collectivité, ALEC et CCRT sont des rôles « territoriaux » : ce qu'ils voient (notamment les demandes) dépend des permissions par réseau ou territoire attribuées par un admin. Un compte de ces rôles **sans permission ne voit rien**.

## Parcours 1 — Inscription publique

Accessible sur [/inscription](/inscription), réservée aux rôles **particulier** et **professionnel**. Il est impossible de s'inscrire soi-même comme gestionnaire ou collectivité.

Règles du formulaire :

- L'acceptation des CGU est obligatoire ; l'abonnement à la newsletter est optionnel.
- Pour un professionnel, la structure et son type sont obligatoires (le type « Autre » doit être précisé). Si un SIRET est renseigné, il est vérifié auprès de l'annuaire officiel des entreprises.
- Un email déjà associé à un compte est refusé, quel que soit l'état de ce compte.

Règles à connaître :

- **Tant que l'email n'est pas confirmé, la connexion est refusée** avec un message explicite.
- Le lien d'activation n'expire pas, mais il est à usage unique.
- Une demande de « mot de passe oublié » sur un compte en attente **active aussi le compte** (la preuve de possession de l'email est équivalente).
- La création du contact dans ADEME Connect (CRM de l'ADEME) est silencieuse : si elle échoue, l'inscription aboutit quand même.

Données modifiées à l'inscription :

| Donnée | Où | Détail |
| --- | --- | --- |
| Email | Base — `users.email` | Normalisé en minuscules |
| Mot de passe | Base — `users.password` | Stocké chiffré, jamais en clair |
| Rôle | Base — `users.role` | `particulier` ou `professionnel` |
| Statut du compte | Base — `users.status` | « En attente de confirmation de l'email » |
| Jeton d'activation | Base — `users.activation_token` | Généré, intégré au lien de l'email |
| Acceptation des CGU | Base — `users.accepted_cgu_at` | Horodatée si cochée |
| Abonnement à la newsletter | Base — `users.optin_at` | Horodaté si coché |
| Entreprise | Base — `users.entreprise` | Fiche complète récupérée via le SIRET |
| Contact CRM | ADEME Connect | Nom, prénom, email, téléphone, rubriques selon rôle et type de structure, consentement newsletter |

Données modifiées à l'activation (clic sur le lien) :

| Donnée | Où | Détail |
| --- | --- | --- |
| Statut du compte | Base — `users.status` | Passe à « valide » |
| Date d'activation | Base — `users.activated_at` | Horodatée |
| Jeton d'activation | Base — `users.activation_token` | Effacé (usage unique) |

## Parcours 2 — Création par un admin

Depuis [Gestion des utilisateurs](/admin/users), un admin peut créer un compte avec **n'importe quel rôle**. Le compte est **immédiatement valide**, sans confirmation d'email.

- Un mot de passe aléatoire est généré : la personne doit utiliser « Mot de passe oublié » pour définir le sien.
- **Un seul cas déclenche un email** : compte créé **actif** avec le rôle **gestionnaire** → email [« Ouverture de l'espace gestionnaire »](/admin/emails?type=auth.gestionnaire.ouverture-espace). Tous les autres rôles ne reçoivent **aucun email** ; c'est à l'admin de prévenir la personne.
- Le contact ADEME Connect n'est créé que pour les rôles gestionnaire, particulier et professionnel.
- Pour un gestionnaire, l'admin choisit aussi ses options de notification : recevoir les nouvelles demandes, recevoir les rappels de demandes en attente (voir [Rôle : gestionnaire](/admin/doc/role-gestionnaire)).
- Événement enregistré : [« Création utilisateur (admin) »](/admin/events?types=user_created_by_admin).

Données modifiées :

| Donnée | Où | Détail |
| --- | --- | --- |
| Compte complet | Base — `users` | Email, rôle, structure, statut « valide », mot de passe aléatoire |
| Options de notification | Base — `users.receive_new_demands`, `users.receive_old_demands` | Choisies par l'admin dans le formulaire |
| Contact CRM | ADEME Connect | Créé uniquement pour gestionnaire, particulier, professionnel |

## Parcours 3 — Création automatique via l'API partenaire

Certains partenaires (ENGIE) synchronisent leurs comptes gestionnaires via l'API. Créations, désactivations et permissions sont pilotées par le partenaire, **sans aucun email envoyé** aux personnes concernées.

Événements enregistrés : [« Création utilisateur (API) »](/admin/events?types=user_created_by_api), désactivation et synchronisation des permissions.

## Ce qui se passe à la connexion

Chaque connexion réussie a des effets de bord :

- Les **demandes déposées avec le même email** (y compris sans compte, via le formulaire public) sont automatiquement rattachées au compte. C'est ce qui permet à un particulier de retrouver une demande faite avant son inscription.
- La date de connexion est mise à jour dans ADEME Connect.
- Événement enregistré : [« Connexion utilisateur »](/admin/events?types=user_login).

Cas de refus :

- Email non confirmé → message explicite demandant de confirmer.
- Compte désactivé par un admin → message générique « Mauvais login/mot de passe », **identique à un mauvais mot de passe**. La personne ne sait pas que son compte est désactivé.

Données modifiées :

| Donnée | Où | Détail |
| --- | --- | --- |
| Rattachement des demandes | Base — `demands.user_id` | Toute demande portant le même email est reliée au compte |
| Date de connexion | ADEME Connect | Mise à jour à chaque connexion |

## Mot de passe oublié

- Si l'email ne correspond à aucun compte actif, **rien n'est envoyé et rien n'est affiché** (protection contre l'énumération d'emails). Une personne au compte désactivé ne recevra donc jamais l'email.
- Le lien reçu est valable <Rule id="passwordResetTokenValidityHours" /> et à usage unique.
- Sur un compte en attente de confirmation, définir un nouveau mot de passe **active aussi le compte**.

Données modifiées :

| Donnée | Où | Détail |
| --- | --- | --- |
| Jeton de réinitialisation | Base — `users.reset_token` | Généré à la demande, effacé à l'usage |
| Mot de passe | Base — `users.password` | Remplacé à la validation du formulaire |
| Statut du compte | Base — `users.status` | Passe à « valide » si le compte était en attente |

## Récapitulatif des emails

| Email | Destinataire | Quand |
| --- | --- | --- |
| [Confirmation d'inscription](/admin/emails?type=auth.utilisateur.confirmation-inscription) | Le nouvel inscrit | À l'inscription publique |
| [Ouverture de l'espace gestionnaire](/admin/emails?type=auth.gestionnaire.ouverture-espace) | Le gestionnaire | Création par un admin, uniquement si compte actif et rôle gestionnaire |
| [Réinitialisation du mot de passe](/admin/emails?type=auth.utilisateur.reinitialisation-mot-de-passe) | L'utilisateur | Demande « Mot de passe oublié » (compte actif uniquement) |

Chaque lien ouvre le modèle correspondant dans [Modèles d'emails](/admin/emails).

## Traçabilité

Chaque étape émet un événement consultable dans [Activité du site](/admin/events) : [« Inscription utilisateur »](/admin/events?types=user_registered), [« Activation utilisateur »](/admin/events?types=user_activated), [« Connexion utilisateur »](/admin/events?types=user_login), [« Création utilisateur (admin) »](/admin/events?types=user_created_by_admin), [« Création utilisateur (API) »](/admin/events?types=user_created_by_api), [« Demande réinitialisation mot de passe »](/admin/events?types=user_password_reset_requested).
