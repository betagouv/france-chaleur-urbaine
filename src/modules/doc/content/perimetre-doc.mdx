# Périmètre et limites

Pour qu'un « non documenté » soit toujours un **choix visible** et jamais un trou silencieux, voici l'état de couverture de cette documentation et le niveau de confiance qu'on peut lui accorder.

Si vous cherchez une règle sur un sujet « Hors périmètre », c'est qu'elle n'existe pas côté métier ; sur un sujet « Partiel », qu'elle n'est pas encore écrite — pas qu'elle n'existe pas.

## Quel niveau de confiance

Tout n'a pas la même garantie de fraîcheur :

- **Les inventaires de la page [Références](/admin/doc/references)** (emails, tâches, statuts, événements) sont **extraits du code** : leur liste et leurs libellés ne peuvent pas être faux ni incomplets.
- **Les valeurs chiffrées soulignées en pointillés** (distances, délais, plafonds) sont lues dans la **même constante que celle utilisée par l'application** : la doc et le code ne peuvent pas afficher deux valeurs différentes. Quelques seuils restent encore rédigés à la main (notamment côté comparateur, dont les hypothèses viennent d'un modèle externe) — ils sont signalés comme tels dans les pages concernées.
- **Le reste est de la prose rédigée**, tenue à jour avec le code par convention d'équipe, et protégée par un test automatique qui bloque la publication si un email, un événement ou une valeur chiffrée n'est plus correctement rattaché. Ce n'est pas une garantie absolue : en cas de doute sur un enchaînement, la source de vérité reste le code.
