# Deployment

## Environments

| Environment | Branch | URL | Database | Purpose |
|------------|--------|-----|----------|---------|
| Local | any | localhost:3000 | Local Docker PostgreSQL | Development |
| Review apps | PR branches | auto-generated by Scalingo | Shared with dev | PR review, testing |
| Staging (dev) | `dev` | (Scalingo dev app) | Dedicated staging DB | Integration testing |
| Production | `main` | france-chaleur-urbaine.beta.gouv.fr | Production DB | Live users |

**Review apps**: Scalingo automatically spawns an environment for each PR. They reuse the staging (`dev`) database — **DB migrations are disabled** on review apps.

**Staging → Production**: merging `main` is ideally a fast-forward from `dev`.

## CI/CD pipeline (GitHub Actions)

Config: `.github/workflows/ci.yml`

On every push:
1. Install dependencies (`pnpm install --frozen-lockfile`)
2. Audit (`pnpm audit`, continue on error)
3. Lint (`biome ci`)
4. Run migrations (against CI PostgreSQL)
5. Run tests (Vitest)
6. Build (`pnpm build` with `GITHUB_CI=true`)

CI services: PostgreSQL 16 + PostGIS 3.5 (user: fcu, port: 5432).

## Hosting: Scalingo

- **Platform**: Scalingo (French PaaS, similar to Heroku).
- **Region**: osc-fr1.
- **App**: france-chaleur-urbaine.

### Buildpacks (order matters)

1. `Scalingo/apt-buildpack` — installs GDAL system libraries.
2. `betagouv/france-chaleur-urbaine-scalingo-buildpack` — compiles Tippecanoe binary.
3. `Scalingo/nodejs-buildpack` — Node.js runtime.

### Required Scalingo env vars (beyond app env)

```bash
LD_LIBRARY_PATH=/app/.apt/usr/lib/x86_64-linux-gnu/blas/:/app/.apt/usr/lib/x86_64-linux-gnu/lapack/
PROJ_LIB=/app/.apt/usr/share/proj
```

### Database migrations on deploy

- **Production and staging**: migrations run as part of the build/start process (`pnpm db:migrate`).
- **Review apps**: migrations are **disabled** (they share the staging DB).

### Rollback

Revert to previous deployment via Scalingo dashboard.

## Background processing

- **Job processor**: started with `pnpm cli jobs start` or `pnpm start:clock`.
- **Cron jobs**: enabled via `CLOCK_CRONS_ENABLE=true`.
- Job types: `build_tiles`, `pro_eligibility_test`, `sync_geometries_to_airtable`, `sync_metadata_from_airtable`.

## Troubleshooting

- **Build fails**: check `pnpm build` locally. Common: missing env vars, type errors, 8GB memory limit.
- **Migration fails**: check `pnpm db:migrate` status. Never force-reset production DB.
- **Tile generation fails**: check Docker (GDAL/Tippecanoe) availability and PostGIS data integrity.
- **Geographic tools**: ensure `USE_DOCKER_GEO_COMMANDS=true` locally, buildpacks on Scalingo.
