# Data Model

PostgreSQL 16 + PostGIS 3.5, 99+ tables. Types generated by `kysely-codegen` into `src/server/db/kysely/database.ts` (single source of truth — read the migration rules before regenerating).

## Orientation
Core: `users` ←(gestionnaire assignment)→ `reseaux_de_chaleur` / `reseaux_de_froid`; `demands` (connection requests, proximity-matched to a network) + `demand_emails`; `pro_eligibility_tests` (+ `_addresses`); `bdnb_batiments`; `jobs` (async queue); `events` (audit). Spatial (PostGIS): `reseaux_de_chaleur` (MultiLineString pipe routes); `zone_de_developpement_prioritaire` / `zones_et_reseaux_en_construction` / `ign_communes` / `zone_a_potentiel_chaud` (MultiPolygon); `*_tiles` (precomputed tiles).

## Kysely (not Prisma)
- Singleton in `src/server/db/kysely/` (pool 5–60, 30 s timeout, app name `FCU-API`). **Never call Kysely from components or tRPC routes — go through services.**
- **Prefer the typed query builder over raw `sql`.** Raw SQL bypasses type checking — use it only when Kysely can't express the operation (PostGIS, `unnest`, advanced CTEs), and verify every table/column against `database.ts` first. Two clean Kysely queries beat one raw query.
- `.select([...])`, never `selectAll()` on production paths (over-fetch). Prevent N+1 (joins / batch, never loop DB calls). Optional filters via `.$if(cond, (qb) => …)`, never string concat.
- Required row → `.executeTakeFirstOrThrow(() => new TRPCError({ code: 'NOT_FOUND', … }))`, not `executeTakeFirst()` + `if (!row) throw`.
- Constant literal projection → `sql.lit<EnumType>('value')` (generic, compile-checked), not `sql<EnumType>\`'value'\`` (silent cast).

## PostGIS
Storage **Lambert 93 (2154)**, I/O **WGS84 (4326)** — transform in the query.
```ts
sql`ST_Distance(ST_Transform(geom, 2154), ST_Transform(ST_SetSRID(ST_MakePoint(${lon}, ${lat}), 4326), 2154))`
sql`ST_Contains(geom, ST_Transform(ST_SetSRID(ST_MakePoint(${lon}, ${lat}), 4326), 2154))`
```

## Naming
snake_case + French legacy names; some columns have spaces/caps (`nom_reseau`, `Identifiant reseau`, `created_at`). FKs `<table>_id`, geometry `geom`, enums stored as text. **Never rename existing columns — it breaks data pipelines.**

- Always use `.select([...])` — never rely on `selectAll()` in production queries (over-fetches columns).
- Prevent N+1: use joins or batch queries, never loop with individual DB calls.
- Use `.$if()` for optional filters rather than building query strings conditionally.
- **Prefer the typed Kysely query builder over raw `sql` template literals.** Raw SQL bypasses type checking and risks referencing nonexistent columns. Reach for raw `sql` only when Kysely cannot express the operation (e.g. PostGIS, `unnest`, advanced CTEs) — and even then, **verify every table/column name against `src/server/db/kysely/database.ts` first**. When two clean Kysely queries are an option versus one raw SQL query, choose the two queries.
- **Never put bare table/column names inside a `sql` template** (`sql\`p.reseau_de_chaleur_ids @> ...\``). Always reference columns through the builder: `eb('col', op, value)`, `eb.ref('alias.col')`, `eb.fn(...)`, or embed a typed sub-query (`sql\`ARRAY(${eb.selectFrom('t').select('id')})\``). The only acceptable raw fragments are the operator/function Kysely lacks (e.g. `ARRAY(...)`, `::int[]`, `<@`, `ANY(...)`) wrapping typed refs. Array/subset checks: `eb('arr_col', '<@', sql\`ARRAY(${subquery})::int[]\`)` rather than `unnest` over a bare column. Existence: `eb.not(eb.exists(eb.selectFrom(...).whereRef(...)))`.
- **Project string/number/boolean literals via `sql.lit<EnumType>('value')` (generic), not `sql<EnumType>\`'value'\`` (template).** The generic constrains the argument so TS rejects values outside the enum at compile time; the template form is a cast and silently accepts anything. Typical use: a `'<discriminator>' AS type` projection in a `UNION ALL`, or a constant column in a CTE.

## Migrations (`src/server/db/migrations/YYYYMMDDHHMMSS_description.ts`)
1. Never edit a migration after it's applied anywhere.
2. **Never run `pnpm db:sync` alone** — it overwrites `database.ts` and wipes manual type refinements (`JSONColumnType<…>`, custom enums, narrowed shapes). Use `pnpm cli db:sync --single <table>` to print one table's regenerated interface, then merge only the diff by hand, preserving every override.
3. Commit the migration + updated `database.ts` together.
4. One `sql` template block per direction (statements separated by `;`) — single round-trip.
5. Always provide both `up` and `down`. Use raw `sql`, never the Kysely schema builder (`db.schema`), in migrations.

Index patterns: `USING GIN (field jsonb_path_ops)` (JSONB), `USING GIST (geom)` (spatial), partial `… WHERE status = 'pending'`.
